## Context

`code/level-structure/enumerate-H.m` (inside `EnumerateH`, ~`:676-678`; the `GetG1plus` path used by `GenerateDataForGerbiestSurjectiveH` builds the same objects at `:40-47`):

```
G1plus:=sub< G | NBOplusgensGL4 >;
// This assertion fails for N = 5, D = 6. Do we understand why?
assert #G/#G1plus eq 2;
```

G = image of Aut_{±μ}(O) ⋉ (O/N)ˣ in GL₄(ℤ/N); G1plus is generated by the GL₄-images of `NormalizerPlusGeneratorsEnhanced(O,mu)` (`polarization-twisting.m:157`, wrapping the hardcoded `NormalizerPlusGenerators`, `elliptic-elements.m`). This is why **N=5 is missing from all generated data** (levels shipped: 1,2,3,4,6).

Empirical fact to explain: fails at N=5, passes at N=3,4,6 (and at 3·1, 3·2 via the level-lifting trick for N=1,2). 5 is the first tested prime ≡ 1 (mod 4) — plausibly the sign/square obstruction that makes the "plus" part surject or collapse mod N differs there (e.g. whether −1 or the norms of AL elements are squares mod N).

## Task

Determine mathematically what [G : G1plus] is for D=6 at N=5 (compute it: is it 1? 4?), why, and what the correct level-N notion of the plus-subgroup is; then fix the code so all N coprime to D work uniformly.
